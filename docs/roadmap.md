# Roadmap

Here is what we are thinking about for future versions. These are ideas we are exploring, not promises.

## Near term

- **Better error messages** — When something goes wrong, clearer feedback about what happened and how to fix it.
- **More supported AI clients** — Adding support for newer tools as they come out.
- **Performance improvements** — Faster startup times and less memory usage.

## Medium term

- **Natural language task planning** — Instead of doing one action at a time, the AI could plan and run multi-step tasks from a single high-level instruction.
- **Workflow automation** — Save and replay common browser workflows.
- **Session recording** — Play back what the AI did for debugging or review.
- **Human-in-the-loop mode** — Sensitive actions like form submission or payment would pause and ask you to confirm before proceeding.

## Longer term

- **Cross-browser support** — Support for Firefox and Edge in addition to Chromium.
- **Scheduled tasks** — Run browser tasks on a timer, like daily reports or monitoring.
- **Test generation** — Describe a test in plain English and let the AI generate and run it.
- **Cloud-optional architecture** — A version that can run in the cloud for teams, and a local-only version for individuals.

## Privacy remains a priority

In all future versions, the core design principle stays the same: your data stays on your machine unless you explicitly choose otherwise.
